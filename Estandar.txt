
Estructura de nombramiento 
	
	Variables: Las variables deberán nombrarse usando notación Camel Case 
		   Deberá evitarse el uso de abreviaciones y caracteres únicos en los nombres de variables además de solo usar nombres descriptivos
		   No se deben utilizar caracteres especiales para el nombramiento de variables
		   Declarar una sola variable por cada línea de código
		
	
	Métodos: Deberán nombrarse usando notación Camel Case
		 Los métodos para operaciones de modificación o consulta de atributos de clase deberán seguir el modelo getter y setter	
		 

	Clases: Deberán nombrarse usando notación Upper Camel Case


	Archivos: Los archivos deberán tener el nombre de su clase principal en Upper Camel Case


	Paquetes: Los paquetes deberán nombrarse con notación Upper Camel Case de manera descriptiva

	
	Constantes: Las constantes deberán nombrarse con todas las letras en mayúsculas y separando cada palabra del nombre con guiones bajos


Estructura de archivos
	
Encabezado (Header)
	- Poner un encabezado completo para que la gente sepa quién es el autor de la clase 

	
		Nombre completo.

		Fecha para saber cuándo se comenzó el desarrollo de la clase DD/MM/AAAA
	
		Fecha de última modificación para saber cuándo fue la última vez que la clase se modifico
	
		Nombre de la clase para saber cómo referirse a éste durante el desarrollo del proyecto o para hacer referencia a él desde otros artefactos

		Propósito de la clase (descripción), para conocer la justificación que hay para la creación de dicho programa.

		/*
		Creador: 
		Fecha de creación: 
		Fecha de última modificación:
		Nombre de clase: 
		Propósito: 
		*/
	
	Comentarios: 
		Utilizarlos únicamente cuando esto sea necesario
		Para justificar porque alguna implementación especifica se implementó de la manera específica en que se hizo
		Para explicar lógica compleja o rara en caso de que esta haya sido necesaria 
		


Indentación y espaciado

	Se debe realizar una Indentación para el código contenido dentro de estructuras de control, métodos y clases
	
	La llave para abrir una estructura de control, método o clase debe encontrarse directamente después de la declaración de esta y únicamente separada por un espacio

	La llave que cierra una estructura de control, método o clase debe encontrarse una línea después a la última línea de código dentro de esta y al mismo nivel de Indentación que su declaración
	
	Se debe usar tabulación en vez de espacio para la Indentación de código

	Los operadores lógicos y paréntesis tanto de apertura como de cierre deben ser precedidos y proseguidos de un espacio simple
	
	Debe haber una línea en blanco dividiendo a cada declaración de métodos dentro de una clase

Buenas practicas

	Se debe evitar declarar atributos de clases como públicos y solo deben ser accesados a través de métodos de la clase

	Se deben utilizar las constantes conforme sea necesario para evitar insertar números o cadenas estáticas directamente al código 

	Los métodos deben tener una sola tarea
	
	Evitar usar variables globales, las variables deberán ser locales siempre que sea necesario

	Evitar el uso de Switch en la medida que sea posible, prefiriendo usar If

	Las variables se deben inicializar siempre que sean utilizadas

	Nunca se debe atrapar una excepción que no haga nada
